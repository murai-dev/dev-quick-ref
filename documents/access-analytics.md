# アクセス解析

## 目的

- デプロイ後に、どのページが見られているかを把握する
- 検索流入キーワードと参照元の傾向を確認する
- 流入の多いカテゴリ・ページを特定し、コンテンツ拡充の判断材料にする
- できるだけサイトの表示速度や体験を損なわない

## 要件

- 静的HTMLにそのまま埋め込めること
- 導入コストが低いこと
- 個人運営でも扱いやすいこと
- まずはページビュー・流入キーワード・参照元の把握ができれば十分
- ローカル開発環境のアクセスを本番集計から除外できること

## 現時点の推奨: Cloudflare Web Analytics

- Cloudflare Pagesとの相性がよく、追加設定なしで有効化できる
- Cookieなし運用を取りやすい
- 軽量で表示速度への影響が小さい

## Search Console との連携

アクセス解析よりも重要度が高い。

- Google Search Console にサイトを登録する
- `sitemap.xml` を送信する
- 「どのクエリで表示されているか」「クリック率は何%か」を毎月確認する
- 流入が多いページを特定し、そのカテゴリのページ追加を優先する
