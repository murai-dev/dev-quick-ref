# MVP candidates

最初の100ページ候補。「エラー解決・困った瞬間の検索」を中心に、カテゴリ別に整理する。

## 選定基準

- 開発者が実務で詰まって検索するフレーズであること
- 英語で検索需要があること
- 静的HTMLページとコードサンプルだけで回答できること
- 「公式ドキュメントを見てください」で終わらないこと

## 調査結果（2026-05-29）

### 調査方法

| # | 指標 | 方法 | 意味 |
| --- | --- | --- | --- |
| 1 | Google Suggest (en/us) | Suggest API `gl=us&hl=en` | Google が「検索需要が高い」と判断しているか（0〜10） |
| 2 | SO top views | Stack Exchange API `/search` intitle マッチ 上位1件の閲覧数 | 同カテゴリの SO 質問がどれだけ見られているか |

> SO top views が 0 の行は「クエリ文字列が SO 質問タイトルと一致しなかった」ケースを含む。需要ゼロを意味しない。

調査スクリプト: `scripts/research.js`
