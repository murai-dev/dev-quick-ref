# 公開後のアクセス増加施策

## 前提

- 静的サイト（Cloudflare Pages）のため、サーバーサイド施策は対象外
- 英語圏の開発者をターゲットにするため、コンテンツは英語のみ
- 「1ページ1問題1答え」という構成を崩さない範囲で施策を打つ

---

## フェーズ1：公開直後にやること（〜1ヶ月）

### 1-1. Search Console と Analytics の設定

- Google Search Console にサイトを登録し、`sitemap.xml` を送信する
- Cloudflare Web Analytics を全ページに埋め込む
- 目的: 最初の流入キーワード・ページを把握し、次の施策の判断材料にする

### 1-2. 技術的 SEO の基礎整備

| 項目 | 内容 |
| --- | --- |
| `sitemap.xml` | 全ページを含む形でビルド時に自動生成する |
| `robots.txt` | クロールを妨げる設定がないか確認する |
| `canonical` | 各ページに自己参照 canonical を設定する |
| OGP / Twitter Card | タイトル・説明・OGP 画像を各ページに設定する |
| Core Web Vitals | PageSpeed Insights で LCP / CLS / INP を計測し 90点以上を目指す |

### 1-3. 各ページのコンテンツ品質確認

各ページに以下が揃っているか確認する。

- Quick Answer セクション（コードブロック + コピーボタン）
- When this happens（1〜2文）
- Related セクション（関連ページへの内部リンク3〜5件）
- カテゴリ索引ページへのリンク

---

## フェーズ2：流入が始まったら改善する（1〜3ヶ月）

### 2-1. Search Console でのキーワード分析

- 「表示回数が多いのにクリック率が低い」ページ
  → `<title>` と `<meta description>` を改善する
- 「想定外のクエリで流入している」ページ
  → そのクエリに対応した内容を追記する（Quick Answer や When this happens を拡充）
- 「ほぼ流入がない」ページ
  → タイトルのキーワードを見直すか、関連ページからの内部リンクを増やす

### 2-2. 高パフォーマンスカテゴリの深掘り

流入が多いカテゴリを特定し、そのカテゴリの問題ページを優先的に追加する。

例: Git カテゴリの流入が多い → Git の問題ページを30 → 50ページに増やす

### 2-3. 内部リンクの改善

- カテゴリ索引ページを充実させる（ページ一覧 + 各ページへのリンク + 概要1行）
- Related セクションのリンク先を Search Console のデータを見ながら最適化する

---

## フェーズ3：ページ数を積み上げる（3ヶ月以降）

- Phase 2 の調査結果を元に、需要があるカテゴリを拡充
- 新しいカテゴリを追加する（候補: Vim、tmux、ffmpeg、sed/awk、Python等）
- 各カテゴリ50〜100ページを長期目標とする

---

## やらないこと

- 短期的なアクセス増加を狙ったSNS投稿（継続コストが高い）
- バックリンク獲得のための外部コンテンツ配置（効果が不確か）
- ページ品質を下げてでも量を増やすこと
